



### 弹道解算

我看到英雄代码是只解算前哨站，分两种情况来计算：

1. 角速度绝对值大于1的话，会根据旋转中心计算一个**假想板**

   1. 获取子弹打到装甲板的时间+从图像发布到我开始进行弹道解算的问题+偏置时间。

   2. 用角速度（固定的，官方文档有给）预测当前位置经历上面时间后，前哨站三个装甲板的位置，并计算三个装甲板离车辆的位置和离假想板的位置。

   3. 计算**假想板**位置的均值。

   4. 用离车辆距离最小的装甲板和假想板去弹道解算出yaw和pitch角，

   5. ```c++
                  Eigen::Matrix2d R_odom2center,R_odom2armor;
                  R_odom2center << std::cos(alpha), std::sin(alpha),
                      -std::sin(alpha), std::cos(alpha);
                  R_odom2armor << std::cos(beta), std::sin(beta),
                      -std::sin(beta), std::cos(beta);
               
                  Eigen::Matrix2d R_center2armor = R_odom2center.transpose() * R_odom2armor;
                  double decision_angle = -std::asin(R_center2armor(0, 1));
      //计算出装甲板相对假想板的相对角度差
      ```

   6. 执行火控`gimbal_msg.fire_advice = std::fabs(decision_angle) * 180.0 / M_PI <= 9.0 ? true : false; `.

2. 角速度绝对值小于1的话

   1. 就直接对yaw角进行预测，直接选当前装甲板就好了。
   2. 但是火控方面变化角度阈值减小了`bool yaw_flag = std::fabs(gimbal_msg.yaw_diff) <= 2.0 ? true : false;bool pitch_flag = std::fabs(gimbal_msg.pitch_diff) <= 2.0 ? true : false;`
   3. 执行火控

```c++
#include<iostream>
#include<cmath>

int main()
{
    int n;
    cin>>n;
    uint32_t temp = n;

    {
        if(temp&1)
        {
            count1++;
        }
        else
        {
            count0++;
        }
        temp<<=2;
    }
    if(count1 > count0)
    {
        cout<<count1<<",1 Win"<<endl;
    }
    else
    {

    }
    return 0;
}
